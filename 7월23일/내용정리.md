## 프로그래밍 기초 HTML · CSS · Javascript

c는 유지보수가 어려눈 절차형 언어, 자바로 바꿀수없나?

클라이언트 : 웹페이지를 요청하는 사용자의 장치 -> 웹브라우저 : 백엔드(rest API+제이슨 포맷이 표준이 되감)

서버 : 요청을 받아 처리하고 데이터를 제공하는 컴퓨터

HTTP : 웹 자원을 주고 받는 약속

URL : tmzla, 호스트, 포트, 경로, 쿼리, 프래그먼트

브라우저의 렌더링 과정 : 웹 브라우저는 서버에서 받은 HTML 문서를 해석하여 웹페이지로 렌더링 : 파싱, 렌더 트리, 레이 아웃, 페인트 : HTML의 작성 순서와 구조가 그대로 화면의 구조가 된다

HTML(구조) -> CSS(표현) -> JavaScript(동작)

Emmet, live server

웹에서 코드가 실제로 어떻게 해석 표시되는지 도구 : elements(화면 출력 제대로 안되면 확인), console(동작 확인), network, application, 기기툴바

HTML : 웹페이지의 구조와 컨텐츠를 정의하는 마크업 언어

DTD(문서형 정의) -> 문서의 시작과 끝 -> 문서 정보(메타데이터) -> (브라우저에 표시할 내용)

태그, 요소(태그 + 내용), 속성(여는 태그 안에 이름=값 형태로 추가 정보 지정)(id, class, style lang, hidden)

텍스트 태그 : 제목과 문단 : <h1>~<h6>, <p> <hr>

    블록 요소와 인라인 요소 : <div><span>차이점

폼 태그<form> : 사용자 입력을 수집하고 서버로 전송하는 양식을 정의

    <input>, label, select, button

    폼 유효성 검사 속성 : 브라우저 기본 유효성 검사를 킬 수 있다 : 받은데이터가 정말 클린한건지, 많이 하면 네트워크 부하, 앞단부터 최대한 밸리데이션 하기

미디어 태그 mig, audio, video

시맨틱 태그와 <div>의 차이

    시맨틱 태그 : 문서의 구조와 의미를 명확히 나타내는 HTML5 태그

        가독성(코드 구조가 명확), 접근성(화면 낭독기가 콘탠츠가 콘텐츠 구조를 더 잘 이해한다), SEO(검색 엔진이 주요 정보를 효율적으로 파악)

    div는 의미가 없는 중립 컨테이너, 적당한 시맨틱 태그가 없을때만 쓴다

CSS : 웹 스타일링

    인라인 스타일(라인하나), 내부 스타일(문서 안), 외부 스타일(플래너가 사용하는 방식)

    선택자(스타일을 HTML 요소들), 속성(스타일을 적용할 특성), 값(속성 값)

    선택자 조합

    가상 클래스와 속성 선택자(:hover)

    우선 순위(명시도)와 캐스케이딩

        여러 규칙이 충돌하면 명시도 점수가 높은 규칙이 최종 반영
        점수가 같으면 나중에 선언된 규칙이 이긴다

    상속 : 부모요소의 스타일 일부가 자식요소에 자동으로 적용
    HTML이 계층적 트리구조 -> CSS계층적(배경색은 상속x)

    색상 표기 : hex, rgb...

    절대 단위, 상대단위(실무(디바이스에따라 화면이 늘고 줄어서))

    박스 모델 : 콘텐츠 패딩 테두리-마진

    타이포그래피 속성 : font-family

    웹폰트와 @font-face, 폴백 체인

    배경, 테두리, 모서리

    그림자와 그라디언트

    CSS 변수

    레이아웃과 일반 흐름

        일반 흐름(아무 설정도 하지 않았을 떄의 기본 배치 규칙)(블록 요소는 위에서 아래로, 인라인 요소는 왼쪽에서 오른쪽으로 쌓인다, 레이아웃이란 이 흐름을 의도적으로 바꾸는 일이다.)

    position 속성

    relative와 absolute 조합

    sticky 헤더

    z-index(요소가 겹칠 떄 누가 위에 그려질지 정하는 속성)(쌓인맥락-자식을 아무리 올려도 이길 수 없다)

    flexbox 개념과 두축

    justify-content 주 축 정렬

    align-items - 교차 축 정렬

    flex-wrap, gap

    grid
